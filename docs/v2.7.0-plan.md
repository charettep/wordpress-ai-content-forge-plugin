# AI Genie v2.7.0 — Frontend Rehaul Implementation Plan

## Overview

A structural redesign of the wp-admin settings page and Gutenberg sidebar, informed by:
- Live analysis of both interfaces via Chrome DevTools
- Professional plugin gold standards: Yoast SEO, WooCommerce, RankMath, WP Forms
- Codex AI's independent audit recommendations

**Core principle:** Reorganise the UI so information is layered (overview → detail on demand), reduce cognitive load, and match WP-native component patterns. No backend logic breaks — all form names, IDs, and REST endpoints are preserved.

---

## Phase 1 — Admin Tab Navigation + Summary Strip

**Goal:** Replace the flat settings dump with a tabbed layout and quick-glance status strip.

### Tasks

1. **Bump version to 2.7.0**
   - `ai-genie.php`: plugin header `Version:` and `AIG_VERSION` constant

2. **Rewrite `render_page()` in `admin/class-aig-admin.php`**
   - Add PHP variables: `$provider_labels`, `$default_provider`, `$default_model` (via `match{}`)
   - Render summary strip (`.aig-summary-strip`) at the top:
     - Cell: Default AI (provider name + model)
     - Cell: Max tokens value
     - Cell: Temperature value
     - Badges row: Claude / OpenAI / Ollama with live status dots (`.aig-summary-badge[data-summary-provider]`)
   - Render tab nav (`.aig-tab-nav.nav-tab-wrapper`) with 4 `<a data-tab="...">` links:
     - Providers | Generation | Prompts | Ollama Setup
   - Render single `<form id="aig-settings-form">` wrapping all 4 tab panels
   - Tab panels (`<div class="aig-tab-panel" data-panel="...">`)
     - `providers`: Claude card, OpenAI card, Ollama card (each with API key / base URL, model select, status span)
     - `generation`: Default Provider radio cards + Generation Defaults (max tokens, thinking tokens, temperature)
     - `prompts`: Prompt Templates (blog, social, product, custom)
     - `ollama-setup`: 8-step Ollama installation wizard (static guide)
   - Sticky save footer (`.aig-save-footer#aig-save-footer`):
     - Dirty notice: `.aig-save-dirty-notice#aig-dirty-notice`
     - Discard button: `#aig-discard-btn`
     - Submit input `#submit`
   - Ollama provider card includes cross-tab link: `<a class="aig-tab-link" data-target-tab="ollama-setup">`

   **Critical: Preserve all existing name/ID/class hooks**
   - `name="aig_settings[claude_api_key]"` etc.
   - `id="status-claude"`, `id="status-openai"`, `id="status-ollama"`
   - `id="aig-max-output-tokens"`, `id="aig-max-thinking-tokens"`, `id="aig-token-limit-hint"`
   - Classes: `aig-api-key-input`, `aig-base-url-input`, `aig-model-select`, `aig-ollama-auth-input`
   - Data attrs: `data-provider`, `data-role="header-name"`, `data-role="header-value"`

3. **Update `assets/js/admin.js`**
   - Add tab navigation: `getInitialTab()`, `activateTab(tabId)`
   - Tab click handler, cross-tab link handler (`.aig-tab-link`)
   - `localStorage` persistence under key `aig-active-tab`
   - `history.replaceState` URL sync (for bookmarks; does not survive settings-save redirect)
   - Dirty state: `isDirty`, `markDirty()`, `markClean()`
   - Form change listener → `markDirty()`
   - Discard button → `form.reset()` + `markClean()` + re-sync all providers
   - Form submit → `markClean()` + save active tab to localStorage
   - Summary badge: `updateSummaryBadge(slug, status)` targeting `[data-summary-provider]`
   - Extend `setProviderStatus()` to call `updateSummaryBadge()`

4. **Update `assets/css/admin.css`**
   - Extend `:root` tokens: `--aig-warning`, `--aig-muted`, `--aig-bg-hover`, `--aig-bg-active`
   - `.aig-summary-strip`: flex row, white card, shadow, `flex-wrap`
   - `.aig-summary-badge` variants: `.is-connected`, `.is-checking`, `.is-error` + `.aig-badge-dot`
   - `.aig-tab-nav.nav-tab-wrapper`: standard WP nav-tab styling, remove bottom gap
   - `.aig-tab-panel { display: none }` + `[aria-hidden="false"] { display: block }`
   - `.aig-save-footer`: `position: sticky; bottom: 0; z-index: 100; backdrop-filter: blur(4px)`
   - `.aig-save-footer.is-dirty` reveals dirty notice and discard button
   - Provider cards, code blocks, guide sections (from Ollama wizard)
   - Responsive breakpoints for mobile (`max-width: 782px`)

## Phase 3 — Prompt Editor Split Layout

**Goal:** Replace 4 always-visible full-height textareas with a left-rail selector + right editor.

### Tasks

1. **Left rail (`.aig-prompt-rail`)**
   - Vertical list of prompt types: Blog Post, Social Media, Product Description, Custom
   - Each item is a clickable "tab" that activates the corresponding editor
   - Active state: left border accent + highlighted background
   - Icon or emoji per prompt type

2. **Right editor pane (`.aig-prompt-editor`)**
   - Shows one textarea at a time based on rail selection
   - Textarea remains in the form (not dynamically removed) — just hidden/shown via CSS class
   - Placeholder tokens listed below editor: `{topic}`, `{keywords}`, `{length}`, `{tone}`
   - Character count indicator

3. **Prompt type icons** (WP dashicons or emoji)
   - Blog: `dashicons-media-document` or ✍️
   - Social: `dashicons-share` or 📱
   - Product: `dashicons-cart` or 🛍️
   - Custom: `dashicons-edit` or ⚙️

4. **JS for rail switching**
   - `activatePromptType(type)` — adds/removes `.is-active` class on rail items and editor panes
   - No form serialization change — textareas stay in DOM

5. **CSS**
   - `.aig-prompt-layout`: CSS grid `200px 1fr`
   - `.aig-prompt-rail-item.is-active`: accent border-left + bg
   - `.aig-prompt-editor-pane { display: none }` + `.is-active { display: block }`

## Phase 4 — OpenAI Model List Filtering

**Goal:** Remove deprecated/non-chat models from the live-fetched OpenAI model list.

### Tasks

1. **Edit `includes/class-aig-provider-openai.php`** (or wherever `sync_provider()` is implemented)
   - After fetching models from OpenAI `/v1/models`, filter to only include:
     - Prefix `gpt-` (text generation)
     - Prefix `o1`, `o3`, `o4` (reasoning models)
     - Exclude: `dall-e-*`, `whisper-*`, `tts-*`, `text-embedding-*`, `babbage-*`, `davinci-*`
   - Sort by model ID descending (newest first)
   - Limit to ~30 models max to avoid dropdown overload

2. **Update `MODEL_TOKEN_LIMITS` in `admin.js`** if new models ship
   - Keep the prefix-match list current

## Phase 6 — Gutenberg Sidebar React Refactor

**Goal:** Full React component restructure of the Gutenberg sidebar panel.

### Tasks

1. **Icon update**
   - Replace emoji/placeholder with `@wordpress/icons` `bolt` icon in panel header

2. **Status chip in header**
   - Small pill component: provider name + connected/error indicator
   - Reads from REST API or `localize_script` data on load

3. **Panel restructure** (using WP `PanelBody`, `PanelRow`)
   - **Content** panel: Topic, Keywords, Length, Tone controls
   - **Options** panel: Provider override (if different from default), Model override
   - **Advanced** panel: Max tokens override, Temperature override (collapsed by default)
   - **Usage + Result** panel: merged — shows last generation stats + preview of generated text
   - Remove ALL_CAPS labels → use sentence case

4. **Sticky Generate button**
   - Float at bottom of sidebar panel
   - Disabled while generating (show spinner)
   - Re-enabled after response

5. **Build system**
   - `docker run --rm -v .:/plugin -w /plugin node:20-alpine sh -c "npm ci && npm run build"`
   - Or local `npm run build` if Node is available
   - Output: `gutenberg/build/index.js` (existing path)

6. **Test**
   - Open Gutenberg editor on any post
   - Verify panel renders, Generate button works, output inserts into editor
   - Verify status chip reflects actual connection state

## Release Process (per CLAUDE.md)

After all phases complete:

1. Confirm `AIG_VERSION` and plugin header both read `2.7.0`
2. Update `README.md` with v2.7.0 changelog
3. Commit all changes, push to `main` on GitHub
4. Run `./scripts/build-release.sh` → produces `ai-genie-v2.7.0.zip`
5. Publish GitHub release for `v2.7.0`, attach zip
6. Verify zip installs cleanly via wp-admin upload UI

## Implementation Approach

Each phase uses the **subagent-driven development** workflow:
1. Implementer subagent writes code + self-reviews
2. Spec compliance reviewer verifies against requirements
3. Code quality reviewer checks structure/maintainability
4. Controller (main Claude) commits and deploys

Deployment via: `cd /home/p/Desktop/ai-genie/ai-genie-plugin && ./scripts/build-release.sh && ./scripts/docker-install-plugin.sh`
